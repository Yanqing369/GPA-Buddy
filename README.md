# GPA-Buddy / 请出题 —— 你的期末复习搭子

> 嗨！如果你正在为期末考试发愁，翻着几十页课件却不知道重点在哪里，那你来对地方了。这个网站叫 **GPA-Buddy**（也叫"请出题"），是一个帮你把课件、笔记、往年试卷一键变成练习题的智能刷题助手。

---

## 🙋 普通用户看这里：这网站能干啥？

咱们不整那些虚的，直接说人话——这个网站主要做三件事：**出题**、**整理**、**刷题**。

### 1. 从资料生成题库：课件扔进去，题目吐出来

你有没有过这种经历：老师给了几百页的 PDF 课件或者 PPT，看完一遍跟没看一样，也不知道会考什么。这时候，你就可以把文件直接拖进「从资料生成」页面。

- **支持的格式**：PDF、Word（DOCX）、PPT（PPTX）、Excel（XLSX/XLS）、TXT
- **想生成多少题**：20 道到 200 道，你自己选
- **语言**：可以选中文或英文出题
- **怎么做到的**：AI 会读你的课件，自动提取知识点，然后生成选择题，每道题都带选项、正确答案和解析

更爽的是，生成过程是**实时流式**的，你不用干等着，题目会一道一道在眼前跳出来，像看直播一样。生成完成后，你可以先预览，满意了再保存到题库去刷。

> 💡 小提示：如果是 PDF 文件，AI 连里面的图表都能"看到"；如果是 Word 或 PPT，会先提取文字再出题。

### 2. 整理现成题库：乱七八糟的试卷，AI 帮你理清楚

很多学校都有"往年试题"这种东西，但格式通常一言难尽——可能是扫描的 PDF，可能是排版混乱的 Word，甚至是从群里复制的 TXT。你把这种文件上传到「整理现成题库」，AI 会自动：

- 识别出题目、选项、答案
- 补全缺失的解析
- 转换成标准格式

就算格式再乱，AI 也会尽力帮你理顺。整理完后同样是保存到题库，随时随地在线刷。

### 3. 在线刷题：像玩游戏一样复习

生成或整理好的题库都会出现在「题库管理」页面。点一下"开始刷题"或"继续练习"，就进入练习模式：

- **答题**：选择题，点选项、提交，立刻知道对错
- **看解析**：每道题都有答案解析，告诉你为什么选这个
- **查来源**：最厉害的是，如果这道题是从你的课件生成的，你可以点"来源"查看它出自 PDF 的第几页、PPT 的第几张幻灯片、Word 的哪个章节，甚至能直接在线预览原始文件！
- **收藏**：遇到重点题或易错题，点星星加入"精选"，下次可以只刷精选题
- **进度记忆**：刷到一半去吃饭？没问题，下次回来"继续练习"会从你停下的地方接着来
- **完成统计**：全部刷完后会弹出成绩，告诉你最终正确率

### 4. 题库管理：你的专属题库

在管理页面，你可以：

- 看到所有题库和练习进度（已完成 / 进行中 / 未开始）
- 显示正确率和上次练习时间
- 把题库导出成 JSON 文件备份，或者从 JSON 导入
- 删除不想要的题库

### 5. 多语言支持

网站支持 **简体中文、繁体中文、英文、韩文** 四种界面语言，会根据你的浏览器语言自动切换，也可以手动切换。

## 🧑‍💻 开发者看这里：技术架构和实现细节

如果你是个开发者，想了解一下这个项目是怎么搭起来的，或者想参与进来，下面是一些技术层面的介绍。

### 项目结构

这是一个**纯前端静态网站**，没有复杂的构建流程，直接打开 HTML 就能跑（当然，AI 生成需要后端 API）。

```
.
├── index.html              # 首页，功能入口
├── DocGenerate.html        # 从资料生成题库（PDF 专用入口）
├── DocGenerate.js          # PDF 生成核心逻辑
├── docgenerate_text.js     # 非 PDF 文件生成核心逻辑
├── organize.html           # 整理现成题库
├── manage.html             # 题库管理
├── practice.html           # 练习模式
├── donate.html             # 捐款页面
├── fileViewer.js           # 源文件预览器（PDF/Word/Excel/PPT/TXT）
├── backend/                # 后端服务目录
├── login/                  # 登录相关
├── resources/              # 静态资源（图片、支付二维码等）
└── wechat-miniprogram/     # 微信小程序版本
```

### 前端技术栈

| 技术 | 用途 |
|------|------|
| **Tailwind CSS** | 全部样式，响应式布局 |
| **Dexie.js** | IndexedDB 封装，本地数据库 |
| **PDF.js** | PDF 文本提取和页面渲染 |
| **mammoth.js** | Word 文档（DOCX）文本提取和 HTML 渲染 |
| **xlsx.js** | Excel 文件读取和表格渲染 |
| **JSZip** | PPTX 文件解压和 XML 文本提取 |
| **pdf-lib** | PDF 页码标记注入 |

### 本地数据存储（IndexedDB）

通过 Dexie.js 管理，数据库版本 5，包含以下表：

- `questionBanks`：题库数据（名称、题目数组、收藏列表、时间戳）
- `practiceProgress`：练习进度（当前题号、答案记录、正确率、完成状态）
- `sourceFiles`：原始文件二进制数据（用于"查看来源"功能）
- `settings`：用户设置

### AI 生成流程

1. **PDF 文件**：用 `pdf-lib` 在每一页注入页码标记（`-------------【filename_pageN】-----------`），然后将处理后的 PDF 发送到后端 `/chat` API。后端调用 Gemini/Vertex AI，让模型根据 PDF 内容生成指定数量的选择题。
2. **非 PDF 文件**：在前端用 `mammoth.js`、`xlsx.js` 或 `JSZip` 提取文本，同样在文本中插入位置标记，然后分批发送到后端 API 生成题目。
3. **整理功能**：类似非 PDF 流程，但 Prompt 不同——不是"生成"题目，而是"识别并整理"已有题目格式。
4. **流式输出**：后端支持 SSE 流式返回，前端实时解析并渲染到页面。

### 源文件查看器（Source Viewer）

这是一个比较用心的功能。练习时每道题可能带有 `source` 字段（如 `【filename_page3】`），点击后会根据文件类型调用不同的渲染器：

- **PDF**：PDF.js 渲染 Canvas，高亮目标页
- **Word**：mammoth.js 转 HTML，按章节高亮滚动
- **Excel**：xlsx.js 转 HTML 表格，支持 Sheet 切换
- **PPT**：JSZip 解压后解析 `ppt/slides/slideN.xml`，提取文本按幻灯片展示
- **TXT**：按 50 行分页高亮显示

### 国际化（i18n）

每个页面内置独立的 `i18n` 对象，支持 `zh`、`zh-TW`、`en`、`ko`。通过 `data-i18n` 属性自动替换文本，语言设置保存在 `localStorage`。

### 后端 API

- 基础地址：`https://moyuxiaowu.org`
- 主要接口：`POST /chat`，接收 messages 数组，返回流式或非流式 JSON
- 人机验证：Cloudflare Turnstile（`0x4AAAAAACyCQq1B6IXFL27N`）

## 💼 投资人看这里：产品价值和发展潜力

### 解决什么痛点？

对学生群体来说，期末复习是最刚需但也最痛苦的场景之一：

- **课件多**：一门课几百页 PPT/PDF，学生没时间逐页细读
- **题库少**：很多老师不给练习题，或者给的题目格式混乱
- **效率低**：传统复习是"看资料→理解→记忆"，而"做题"是最有效的主动学习方式，但学生往往没题可做

GPA-Buddy 把这个流程压缩成了：**上传课件 → AI 出题 → 在线刷题 → 追踪薄弱点**。本质上是用 AI 把"静态资料"变成了"交互式学习工具"。

### 核心竞争优势

1. **"来源可查"是差异化亮点**
   市面上很多 AI 出题工具只能生成题目，但无法对应回原始资料的位置。GPA-Buddy 在文件预处理阶段就注入了位置标记，让每道题都能追溯到课件的具体页码/幻灯片。这对学生复习极有说服力——做错的题，可以立刻回到原文去补漏。

2. **完全本地化的数据隐私**
   题库、练习记录、甚至原始文件都存储在用户的浏览器 IndexedDB 中，不需要注册账号就能使用（虽然也有登录功能）。这对注重隐私的学生来说非常友好。

3. **公益模式，用户零门槛**
   网站明确标榜"公益模式免费网站"，没有付费墙，没有订阅制。这种策略在学生群体中传播极快，天然具备口碑裂变潜力。

4. **多语言适配，国际化潜力**
   除了中文，还完整支持英文和韩文界面，说明产品从一开始就有服务国际学生的打算，尤其是留学生和海外中文学习者市场。

### 商业模式与可持续性

目前采用的是**捐赠驱动**模式：

- 每生成一次题库，API 调用成本约为 0.5 元人民币/港币
- 网站通过 banner 和完成页面引导用户捐款
- 支付方式覆盖了中国内地（支付宝/微信）、香港（AlipayHK/FPS）的常用渠道

**潜在的商业化路径**：

- **Freemium 模式**：免费用户有每月生成次数限制，付费订阅解锁无限生成
- **B2B 校园合作**：与学校、辅导机构、MOOC 平台合作，提供批量出题 API
- **增值功能**：比如错题分析、知识点图谱、多人班级排行榜等
- **微信小程序**：已有 `wechat-miniprogram` 目录，说明正在布局微信生态

### 技术资产与扩展性

- **前端完全静态化**，部署成本极低（GitHub Pages / Cloudflare Pages 即可）
- **后端 API 解耦**，可灵活切换不同 LLM 供应商（当前 Gemini，未来可接 Claude、GPT-4o、国产大模型）
- **文件解析能力自研**，对 PDF/Office/PPT 的前端解析积累了完整方案
- **IndexedDB 本地存储架构**让产品可以做成 PWA 离线应用，进一步降低服务器成本

### 用户群体与市场规模

目标用户非常清晰：

- 大学生（期末复习、考研备考）
- 高中生（文科背诵类科目）
- 职业考试人群（CPA、法考、教师资格证等）
- 语言学习者（上传阅读材料生成理解题）

全球高等教育学生数以亿计，只要产品力够强，靠自然流量和学生社群传播就能快速起量。
