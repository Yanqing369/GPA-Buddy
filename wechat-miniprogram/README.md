# 微信小程序 - 后端连接检测

用于检测微信环境下能否访问后端服务 `moyuxiaowu.org`

## 文件结构

```
wechat-miniprogram/
├── app.js              # 小程序入口
├── app.json            # 全局配置
├── sitemap.json        # 站点地图
├── pages/
│   └── index/
│       ├── index.js    # 页面逻辑
│       ├── index.wxml  # 页面结构
│       └── index.wxss  # 页面样式
└── README.md           # 说明文档
```

## 使用方法

### 1. 导入项目

1. 打开微信开发者工具
2. 选择 "导入项目"
3. 选择 `wechat-miniprogram` 文件夹
4. 填写 AppID（如果没有，选择测试号）
5. 点击 "导入"

### 2. 配置合法域名（重要）

#### 开发环境
在开发者工具中：
- 点击右上角 "详情"
- 勾选 "本地设置" → "不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"

#### 生产环境
需要在微信公众平台配置：
1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入 "开发" → "开发管理" → "开发设置"
3. 在 "服务器域名" → "request 合法域名" 中添加：
   ```
   https://moyuxiaowu.org
   ```

### 3. 运行测试

1. 点击 "开始检测" 按钮
2. 查看检测结果：
   - ✅ 绿色：测试通过
   - ⚠️ 黄色：警告但可继续
   - ❌ 红色：测试失败

## 测试项目

|--------|------|

| CORS | 跨域支持检测 |

## 常见问题

### 1. 提示 "fail url not in domain list"

**原因**: 域名不在白名单中

**解决**:
- 开发环境：勾选 "不校验合法域名"
- 生产环境：在微信公众平台添加域名

### 2. 提示 "fail ssl hand shake error"

**原因**: SSL 证书问题

**解决**:
- 检查后端 HTTPS 证书是否有效
- 确保证书链完整

### 3. 提示 "fail timeout"

**原因**: 请求超时

**解决**:
- 检查网络连接
- 确认后端服务正常运行
- 检查防火墙设置

### 4. 真机调试失败但模拟器正常

**原因**: 真机有域名白名单限制

**解决**:
- 确保已在微信公众平台配置合法域名
- 使用 "真机调试" 模式时开启 "不校验域名"

## 修改后端域名

如需测试其他域名，修改 `app.js`：

```javascript
globalData: {
  backendUrl: 'https://你的域名.com'
}
```

并修改 `pages/index/index.js` 中的所有请求 URL。

## 注意事项

1. 微信小程序要求使用 **HTTPS**
2. 生产环境必须在微信公众平台配置 **request 合法域名**
3. 不支持 IP 地址和 localhost
4. 域名必须完成 ICP 备案（国内服务器）
